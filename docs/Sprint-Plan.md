# UniTrack BD - Sprint Plan

## Project Overview

**Project Name:** UniTrack BD – University Bus Tracking & Automated Payment System

**Methodology:** Scrumban

**Sprint Duration:** 1 Week

**Total Sprints:** 6

---

# Sprint 1 – Project Foundation & Authentication

**Goal:**
Establish the project foundation and prepare the development environment.

### Objectives
- Setup GitHub repository and branching strategy
- Setup React Native and React.js projects
- Setup Node.js + Express backend
- Configure PostgreSQL database
- Create database schema
- Implement JWT Authentication
- Student Login & Signup UI
- Basic Navigation
- Prepare reusable UI components

### Deliverables
- Project structure ready
- Authentication working
- Database connected
- Initial frontend screens completed

---

# Sprint 2 – Student Application Core

**Goal:**
Develop the core Student application.

### Objectives
- Student Dashboard
- Live Bus Tracking UI
- Google Maps integration
- Bus ETA display
- Student Profile
- Authentication integration with backend
- Route & Schedule view

### Deliverables
- Functional Student application
- Live map integrated
- User authentication connected

---

# Sprint 3 – Wallet & QR Payment System

**Goal:**
Develop the digital wallet and QR-based boarding system.

### Objectives
- Wallet module
- bKash wallet top-up (or sandbox/mock integration)
- QR Code generation
- QR Scanner integration
- Automatic fare deduction
- Transaction history
- Payment APIs

### Deliverables
- Complete wallet system
- QR boarding workflow
- Payment records

---

# Sprint 4 – Helper Application

**Goal:**

### Objectives

**Goal:**

---

# Sprint Review

At the end of each sprint the team will:

- Review completed work
- Demonstrate implemented features
- Resolve blockers
- Update backlog
- Plan the next sprint

---

# Definition of Done (DoD)

A task is considered completed only when:

- Code is implemented.
- Code is committed to GitHub.
- Pull Request is reviewed and merged.
- Basic testing is completed.
- Documentation is updated.
- Trello card is moved to **Done**.
